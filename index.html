System Design

Multi-Agent Architecture

The AI learning coach uses a multi-agent pipeline with a parallel validation layer ensuring every response is grounded, compliant, and on-tone before reaching a learner.

Frontend Layer
Streamlit Chat UI — Field-facing Q&A interface
Orchestration Layer · Python Backend
🎯 Orchestrator Agent — Receives query · Routes through pipeline · Returns final answer
↓
🔍 Semantic Search Agent — Embeds query · Retrieves top KB chunks · Identifies domain
↓
💬 Responder Agent — Drafts answer using ONLY retrieved KB chunks
↓ Parallel Validation
✅ Grounding Validator (the skeptic) — Only KB content? · All names correct? · No labels invented?
🛡️ Compliance Validator (the rule-keeper) — No off-label claims? · Approved language only?
🎵 Tone Validator (the empath) — Coaching voice? · Not robotic/clinical? · Field-appropriate?
⚠️ Refinement Loop — Feedback sent to Responder · Max 2 retries · Fallback message
or
✅ Approved Response — Passed through Orchestrator · Delivered with confidence score
